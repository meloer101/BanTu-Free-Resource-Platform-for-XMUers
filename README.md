## 一、项目介绍

BanTu 是一个面向**厦门大学（XMU）**师生的**开源学习与资源交流平台**。名字来源于 Asen 和 fufu 的同名曲；项目受[浙江大学](https://github.com/QSCTech/zju-icicles)、[清华大学](https://github.com/zycheiheihei/THU-CST-Cracker)等高校类似开源资源平台的启发而诞生。我们同时致敬每一位在 GitHub 上开源自己项目的开发者和组织。

本平台由 XMU 学生自发维护，旨在汇集 XMUers 共同维护的**学习资料、生活经验、工具推荐**，乃至游戏攻略等一切真正有用的内容，以**开源的方式**自由流动。我们相信：好的教育资源不应该被反复「重新发明」。

如果想下载部分文件夹，可使用 https://minhaskamal.github.io/DownGit/#/home

## 二、项目理念

### 核心信念

> **把找资源、搭环境、整理笔记的时间省下来，去做真正有意义的事。**  
> 哪怕只是多睡一小时、陪家人吃顿饭、读一本闲书。

时间是唯一不可再生的资源，而**知识应该被共享，而不是被反复浪费**。

### 关于「为什么要分享」

很多人会问：自己辛辛苦苦整理的资料，为什么要外传给别人？

- **信息的特性**：信息是一种**非竞争资源（non-rival resource）**。分享给他人，你自己并不会少任何东西；一个人使用它，并不会减少另一个人使用它的能力。
- **开源不是道德绑架**：分享是一种选择。但分享往往会让资源不断被补充、纠错、更新——一个人的笔记，可以慢慢变成一个**不断进化的知识库**。
- **知识的复利**：分享不是「把成果送出去」，而是把它放进一个更大的循环里。一个人的努力，可以变成一群人的积累；**越分享，生态越繁荣；生态越繁荣，个体反而越受益。**

在开源文化里，真正被重视的是**创造资源的人**——整理资料的人、写笔记的人、维护仓库的人。就像 GitHub 的世界：被尊敬的不是下载代码的人，而是 **commit 代码的人**。

我们想传递的，不是「大家把资料都交出来」，而是一种邀请：

> 如果你愿意，让你的努力帮助更多后来的人。也许某一天，你也会被别人的努力帮助。

您可以选择在自己的资料中不公开署名，也可以选择公开署名。我们也非常期待您在资料中署名，这样我们可以在贡献者名单中看到您的名字，并感谢您的贡献，您更可以在资料中添加使用方法，对使用者的提醒，以及对使用者的祝语等等。

当然如果您不想署名，但是资料（尤其是小组作业这种PPT）中有自己的名字懒得去掉，也可以发在群中由我们进行处理，然后您检查即可。

## 四、如何对项目作出贡献

我们欢迎并非常感谢任何在校同学（尤其是 XMU 师生）的贡献。

### 贡献资料（上传资源）

1. **Fork 本仓库**  
   在 GitHub 上 Fork [BanTu-Free-Resource-Platform-for-XMUers](https://github.com/meloer101/BanTu-Free-Resource-Platform-for-XMUers)。

2. **按规范放置文件**  
   将你的资料（如课程 PPT、笔记、往年试卷等）按仓库内已有的**目录规范**放入对应院系/课程目录。

3. **提交 Pull Request（PR）**  
   提交 PR 并附上简要说明（如：资料名称、适用课程、格式说明等）。维护者审核通过后会合并，你的名字会被记录在贡献者名单中。

### 贡献代码或站点

- 本站前端（HTML/CSS/JS）与仓库一起开放。若你希望改进页面、修复 bug 或增加功能，同样欢迎通过 **Fork → 修改 → PR** 的方式参与。

### 注意事项

- **请勿上传**老师或学校明确规定不能外传的资料。我们既不希望好心办坏事，也不希望平台因此受到影响。
- 上传前请确认不侵犯他人版权；推荐使用自己整理、可自由分享的内容。
